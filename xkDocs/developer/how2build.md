不管你是计算机专业的同学还是非科班的学生，不知道你是否好奇，这样一个网站是如何搭建的？原来凭着一个学生自己的力量，也能构建一份这样的网站。是的，当然可以。和那些大规模的网站相比，只有规模不同。

## 需要攻克的难点

### 爬虫编写

有个东西叫爬虫，从互联网中爬取信息。上到 AI 公司、下到黑灰产，都涉及到爬虫。这个网站也需要爬虫技术的支持，从而定期从 1 系统中获得最新的课程信息。

爬虫可以用 `Python` 的 `requests` 库，需要先定位需要访问的 API 地址，进行访问并接收返回的数据。
有关于 API 的理解与分析，可以看看[这篇文章](https://blog.xialing.icu/2025/02/crawl-dola-xiaoquan/)，有关哆啦校圈的各种 API。
这篇文章的价值在于说明，哆啦校圈的匿名并不是绝对的匿名，所以匿名发癫的人，事实上是可以定位到的。

不幸的是，1 系统的 API 访问需要权限。

### 登录与权限获取

为了确保安全，使用 1 系统的 API 需要获得权限。换句话说，我们必须登录才能使用 1 系统。问题是，怎样让自动化的脚本/爬虫程序能模拟人类登录的过程，从而访问需要权限才能访问的 API？这需要我们理解 1 系统登录基于的 OAuth2 架构（其实不了解也行）。
关键在于理解一连串的 API 调用，在代码中模拟这个过程。

具体的分析过程不再赘述，可以参考[这篇文章](https://blog.xialing.icu/2025/01/tongji-login/)。写的有些流水帐，感兴趣的读者可以在读文章的时候亲自去调试。登录相关的代码在[仓库](https://github.com/XiaLing233/tongji-course-scheduler/blob/master/crawler/utils/loginout.py)中。

### 数据组织

把数据爬取下来后，需要用组织化的方式存储，也就是数据库。关系型数据库是**数据库系统原理**这门课要讲述的内容，不过这门课重理论，真要实践，还是要亲自做一些小工具玩玩。设计数据库的结构是本项目的重点，因为一个合理的数据组织方式会对后续的后端代码编写带来很大的便利。

> 说句实话，当前项目的数据库组织并不是很完美。例如，通过点击时间表检索课程基于字符串匹配，而不是时间匹配。

### 网站搭建

网站有 3 大部分：前端、后端、数据库。如果利用服务器来部署网站，可能需要 nginx 来进行反向代理，也就是当用户输入链接访问网站时，需要根据链接把服务器中对应的文件返回给用户。这个没什么好讲的，不少计算机专业的学生最后就会从事前端开发或者后端开发等岗位。

### 自动化

使用自动化程序可以方便开发者。本项目使用的自动化有：

* `crontab` 定时器任务，每周六 7:00am 进行数据的同步；
* `CI/CD` 流水线(Github Actions)，每当前端/后端的代码更新后，自动同步新的内容到服务器。

> 说到 `CI/CD`，得感谢软件工程这门课的项目，让我体会到流水线真香！不过说到流水线这个词，总是让计科人想起被硬件组支配的恐惧。
